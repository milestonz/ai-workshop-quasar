### **주요 원인들**

1. **브라우저 기본 스타일의 강력한 우선순위**
   - 브라우저가 `<h1>`, `<h2>`, `<h3>` 등에 기본 마진을 적용
   - `margin-block-start: 1em`, `margin-block-end: 1em` 등이 기본값으로 설정
   - 일반적인 CSS `!important`로도 덮어쓰기 어려움

2. **CSS 우선순위 문제**
   - 브라우저 기본 스타일이 사용자 CSS보다 우선순위가 높음
   - 단순한 클래스 선택자로는 기본 스타일을 완전히 덮어쓸 수 없음

3. **`all: unset`의 부족**
   - 처음에는 `all: unset`만 사용했지만, 일부 브라우저에서 완전히 작동하지 않음
   - 추가적인 명시적 속성 설정이 필요했음

### **해결 방법**

1. **`all: unset` + 명시적 속성 설정**

   ```css
   all: unset !important;
   margin: 0 !important;
   margin-block-start: 0 !important;
   margin-block-end: 0 !important;
   ```

2. **JavaScript 인라인 스타일 적용**

   ```javascript
   element.style.setProperty('margin', '0', 'important');
   ```

   - 인라인 스타일이 CSS보다 우선순위가 높음
   - `setProperty`로 `!important` 적용 가능

3. **실시간 마진 제거**
   - HTML 렌더링 후 `nextTick`으로 마진 제거
   - 동적으로 생성되는 요소에도 적용

### **결론**

**브라우저의 기본 스타일이 예상보다 강력했고, CSS만으로는 완전히 덮어쓸 수 없어서 JavaScript 인라인 스타일을 추가로 적용해야 했습니다.** 이는 웹 개발에서 자주 발생하는 문제로, 브라우저 기본 스타일과의 싸움입니다! 😅
